# Sift 宣发文案

## 主标题

别再整理资料了，把它们丢给 Sift。

## 副标题

Sift 会帮你保存、读懂、关联和融合每天遇到的信息，并在你需要时给出可直接使用的内容。

## 一句话定位

Sift 是一个 AI 信息消化器：你负责收集，AI 负责分析、关联和融合，最后把信息变成你能直接使用的内容。

## 核心传播句

> 你不用整理。Sift 会把你保存的信息，分析成以后能直接使用的内容。

备选：

- 把有用的东西先丢进来，Sift 会在后台替你消化。
- 不要再维护知识库了，让 AI 帮你把信息变成答案。
- 保存只是开始，Sift 负责让信息重新派上用场。
- 你负责看到和保存，Sift 负责读懂、关联、融合和给出结果。

## 长文版

# 别再整理资料了，把它们丢给 Sift

你是不是也这样：

刷到一篇文章，觉得以后会用，先收藏。

看到一张截图，里面有个观点很重要，先存相册。

朋友发来一段话，挺有启发，先转发给自己。

临时想到一个产品点子，随手写进备忘录。

看完一个视频，觉得里面有几句话值得回头再看，但最后什么也没记。

当下你觉得：“这个以后肯定有用。”

但过几周再看，它可能已经散落在浏览器收藏夹、微信收藏、相册、备忘录、稍后读、聊天记录和各种文档里。

你知道自己保存过。

但你找不到。

就算找到了，也很难直接用。

这就是很多人的信息困境：

我们并不缺保存工具，缺的是让信息重新派上用场的能力。

## 收藏不是复用，整理也不是答案

大多数工具解决的是第一步：把东西存下来。

但真正的问题往往发生在后面：

这篇文章讲了什么？

这张截图里的观点能不能被搜索？

这条资料和我之前保存过的内容有没有关系？

以后写文章、做研究、做产品判断时，它能不能被重新拿出来？

AI 回答问题时，能不能引用我自己的资料，而不是凭空生成一段看似合理的话？

传统笔记工具通常会把这些问题还给用户：

你要起标题。

你要分类。

你要打标签。

你要整理文件夹。

你要建立知识体系。

你要定期回顾。

听起来都对，但问题是：很多时候，我们保存一条信息的时候，根本还没想清楚它是什么。

它可能只是一句触动你的表达。

可能是一张暂时看不懂但觉得重要的图。

可能是一个还没有成形的判断。

可能是未来某篇文章、某个产品、某次决策的种子。

如果保存之前就要求你想清楚标题、分类和用途，很多信息在第一步就已经流失了。

而如果保存之后还要你自己整理，那它最后大概率还是会堆在那里。

## Sift 想做的事情：不是让你整理，而是替你消化

Sift 是一个 capture-first 的 AI 知识助手。

它的第一原则不是让你更努力地管理知识，而是尽量不让你整理资料。

你看到一个以后可能有用的东西，不需要先想清楚它属于哪个主题，也不需要立刻整理成一篇完整笔记。

你只需要先丢进 Sift。

可以是一条链接。

可以是一段文字。

可以是一张截图。

可以是一组图片。

可以是一句备注。

也可以是“我现在还说不清，但我觉得它重要”。

Sift 会先把原始资料保存下来，然后在后台慢慢处理：

提取正文。

识别图片文字。

分析观点、事实、例子和问题。

找出和旧资料的关系。

发现重复、互补和冲突。

融合成可以被搜索、问答、写作、研究和 Agent 调用的内容。

所以 Sift 的核心体验不是：

“我今天要整理知识库。”

而是：

“这个东西可能有用，先丢给 Sift，后面让 AI 帮我消化。”

## 真正的价值在保存之后

如果只是保存，Sift 和收藏夹没有本质区别。

Sift 真正想解决的是保存之后的事。

一条资料进入 Sift 后，它不应该永远躺在角落里。它应该逐渐变成几种更有用的东西：

一份可追溯的来源。

一个可以继续生长的知识页。

一组能被搜索和问答召回的片段。

一个未来写作、研究、判断和 Agent 工作流可以调用的上下文。

更重要的是，当你真的提出问题时，Sift 不只是把资料列表翻出来给你看。

它应该帮你把相关资料召回、对照、分析和融合，然后给出最合适的内容。

比如你最近连续保存了几篇关于 AI Agent 的文章。

Sift 不只是把它们列在收集箱里，而是应该慢慢帮你看到：

哪些资料讲的是同一个问题。

哪些观点互相补充。

哪些来源可能重复。

哪些判断之间存在冲突。

哪个主题值得沉淀成长期页面。

以后你问“企业应该怎么选 Agent 产品”时，哪些资料可以被引用回来，并融合成一份能直接用于决策或写作的回答。

这才是 Sift 和普通收藏工具的区别。

收藏夹负责“放在那里”。

Sift 负责“让它以后还能被用上”。

## Sift 不是要你变成知识管理专家

很多人放弃知识管理，不是因为懒，而是因为那套流程太重。

他们不是没有想法。

不是没有素材。

不是没有输出需求。

而是每次打开工具，都像在面对一个还没整理好的仓库。

Sift 不想把用户训练成知识管理专家。

Sift 更像一个 AI 信息消化器：

你负责看到、想到、保存。

它负责提取、分析、关联、融合、索引和引用。

你可以粗糙一点。

可以先保存半成品。

可以不分类。

可以不打标签。

可以不知道这条信息未来会用在哪里。

因为真正重要的不是你有没有维护一个漂亮的知识库。

真正重要的是：当你需要写作、研究、判断、问答或让 Agent 工作时，Sift 能不能把你保存过的材料变成可用内容。

## 为什么现在需要这样的工具

AI 让生成内容变得很容易。

但越是这样，属于你自己的资料、判断、经历、阅读痕迹和思考过程，反而越重要。

没有个人知识底座，AI 很容易变成一个只会说通用正确答案的助手。

它不知道你看过什么。

不知道你保存过什么。

不知道哪些来源对你重要。

不知道你过去的判断是怎么形成的。

Sift 想做的是另一层东西：

不是替代所有 AI 工具。

不是做一个通用 Agent。

也不是重新发明一个复杂笔记系统。

Sift 更像你的长期知识层。

它把你每天收集到的资料，变成可以被搜索、被引用、被问答、被外部 Agent 调用的上下文。

当你写文章、做研究、准备方案、判断产品方向，或者让 Agent 帮你完成更复杂任务时，Sift 提供的是那块最容易被忽略、但最关键的东西：

你自己的材料。

## Sift 的一句话

Sift 把你每天看到、想到、收藏的信息，变成以后真正能用的内容。

更简单一点说：

你不用整理。Sift 会把你保存的信息，分析成以后能直接使用的内容。

## 适合谁用

如果你只是偶尔收藏几篇文章，可能暂时不需要 Sift。

但如果你每天都在高强度接触信息，并且这些信息最终会变成某种产出，Sift 就会很适合你。

比如：

写作者、自媒体人、研究者、产品经理、创业者、投资人、咨询顾问、学生，或者任何需要长期积累材料、形成判断、持续输出的人。

你不缺信息。

你缺的是一个地方，让这些信息不要散掉，并且在未来真正用得上。

Sift 想成为这个地方。
