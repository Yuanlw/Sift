# Sift 产品战略

> 写于 2026-05-09。基于当前 MVP 完成度、市场环境和真实用户反馈。

---

## 一句话定位

**Sift 是高强度信息消费者的 AI 第二大脑：你负责收，AI 负责读懂、关联、沉淀，并在你需要时直接给答案。**

---

## 产品目标

### 近期目标（0 → 有真实用户留存）

让 Sift 从"能用的 PC 工具"变成"手机上随手就能扔进来"的日常习惯。

核心指标：**用户连续 14 天每天至少保存一条内容。**

这个数字代表习惯建立，是一切商业化的前提。没有留存，付费转化无从谈起。

### 中期目标（有留存 → 开始收费）

- 推出个人订阅，用 AI 额度封装免费层门槛
- 开放公开注册，补全基础账号体系（邮箱验证、密码找回）
- 让 100 个付费用户成为真实的信息消化日常的一部分

### 长期目标（收费 → 规模化）

- 做团队/组织版，切入研究机构、投资机构、咨询公司、自媒体工作室
- 把 Agent API / MCP 做成独立产品线，成为 AI Agent 的私有知识上下文层
- 成为"AI 时代的个人知识基础设施"，而不只是一个笔记 App

---

## 商业价值

### 为什么有机会

**信息消费量在增长，但复用率接近零。**

普通知识工作者每天在 X、微信、Newsletter、YouTube、播客里接触大量内容，真正被沉淀、复用的不到 1%。现有工具解决的是"存下来"，没有解决"用起来"。

Sift 要做的不是第 N 个收藏工具，而是第一个真正帮用户把信息变成资产的工具。

### 目标用户画像（优先级排序）

**第一批：自己就是用户的内容生产者**

- 在 X 上大量浏览、截图、保存的创作者
- 独立研究者、分析师、写作者
- 痛点：内容散落各处，写作时找不到，AI 问答没有自己的上下文

特征：愿意尝鲜，能接受早期产品，会主动传播。

**第二批：有信息管理刚需的职业用户**

- 投资人（研报、行业信息、公司动态）
- 咨询顾问（客户行业资料、方法论积累）
- 产品负责人（竞品信息、用户反馈、行业趋势）

特征：有付费意愿，需要可靠性和隐私保障。

**第三批：团队级用户**

- 研究团队共享知识库
- 自媒体工作室共享素材库
- 小型咨询/投资团队的私有知识层

特征：ARPU 高，流失率低，是真正的规模收入来源。

### 变现路径

```
个人免费层（AI 额度有限）
    ↓
个人订阅 ¥68–128/月（无限额度 + 高级功能）
    ↓
团队订阅 ¥298–698/月/团队（多成员 + 共享知识库 + 权限管理）
    ↓
Agent API 授权（按调用量或按席位，面向开发者和企业）
```

**关键定价逻辑**：免费层用 AI 处理额度做门槛，而不是用功能做门槛。用户能感受到产品核心价值，但处理量受限——这比功能阉割更自然，也更难绕过。

### 竞争壁垒

| 竞品 | 弱点 | Sift 的机会 |
|---|---|---|
| Notion AI | 整理负担重，保存摩擦高 | capture-first，零整理压力 |
| Readwise Reader | 只做阅读高亮，不做 AI 沉淀 | 自动分析、关联、生成知识页 |
| Mem.ai | 留存差，定价高，市场教育失败 | 更聚焦，Agent API 差异化 |
| Apple Notes / 微信收藏 | 无 AI 处理，无语义检索 | 全链路 AI 消化 |
| ChatGPT Memory | 只记对话，不管外部内容 | 接管用户自己的信息流 |

**最真实的护城河不是功能，是数据飞轮**：用户在 Sift 里存的内容越多，知识关联越丰富，迁移成本越高，问答质量越好。这是时间积累出来的壁垒，钱买不到。

## 现在必须重点解决的三件事

### 第一件：移动端 capture 入口（最高优先级）

**这是 Sift 从"能用"到"必需品"的唯一门槛。**

用户最需要保存内容的时刻在手机上——刷 X、看微信、刷抖音——但 Sift 是 PC 产品。这个错位是当前最大的产品漏洞，直接导致习惯无法建立。

用户的真实场景：
> 在 X 上刷到一篇好文章，截了图，放进相册。三周后完全忘了，永远不会用到。

**解法，按优先级排序：**

**① iOS Share Extension（最小可行，最高价值）**

在 X、Safari、微信里长按链接或图片 → 分享 → 选择"保存到 Sift"→ 内容进 inbox。

整个动作 3 秒，无需打开 App。这一个功能，价值超过后续十个 PC 功能。

**② 移动端 Web 优化**

把 `/inbox` 的 Capture Composer 做到在手机浏览器里丝滑可用：支持快速粘贴链接、发送截图、语音转文字。添加到 iOS 主屏桌面后接近原生 App 体验。

**③ iOS Shortcuts 快捷指令（过渡方案，现在就能做）**

用 Sift Agent API 做一个 Shortcuts 动作，在手机上"分享 → 运行快捷指令 → 进 Sift"。不需要 App 审核，发布快。可以先给早期用户用，验证习惯。

**④ 原生 iOS App（中期，不急）**

独立 App 成本高，App Store 审核周期长。先用前三步建立习惯，有留存数据后再决定是否投入。

---

### 第二件：账号体系补全（公开收费的前提）

当前 Sift 的账号体系"对个人和受控部署够用，对公开 SaaS 不够"（项目评审原文）。

缺的不多，但每一项都是硬门槛：

- **邮箱验证**：防止垃圾注册，Stripe 审核必须
- **密码找回**：没有这个不敢公开注册
- **登录安全日志**：用户信任的基础
- **公开注册流程**：现在 `SIFT_ALLOW_PUBLIC_SIGNUP=false`，开放前需要完整测试

这些不是加功能，是开门的钥匙。没有这几项，Stripe 审核不会通过，用户不敢注册。

建议：把账号体系补全作为一个独立迭代，专门处理，不要和功能开发混在一起。

---

### 第三件：用户留存验证（比获客更重要）

在花任何时间做分发、做增长之前，必须先回答一个问题：

**用户用了 14 天之后还在用吗？**

如果留存率低，增长只是在往漏桶里倒水。

当前要做的不是推广，而是找 10–20 个真实目标用户（写作者、研究者、内容创作者），让他们每天用，观察：

- 他们保存了什么？
- 什么情况下他们会去看 wiki？
- 什么时候他们会用 Ask？
- 他们在哪个环节停下来？

这些答案决定接下来做什么功能，比任何产品规划都重要。

## 路线图建议

```
现在
 ├─ 部署上线（VPS + Docker + Cloudflare）
 ├─ iOS Shortcuts 快捷指令（过渡移动方案）
 └─ 找 10 个真实用户，观察留存

1–2 个月
 ├─ iOS Share Extension（移动 capture 核心入口）
 ├─ 移动端 Web 优化（/inbox 手机体验）
 └─ 账号体系补全（邮箱验证、密码找回）

2–4 个月
 ├─ 开放公开注册
 ├─ 上线个人订阅（Stripe）
 └─ 把前 100 个付费用户的反馈做进产品

4 个月以上
 ├─ 团队版（多成员、共享知识库）
 ├─ Agent API 独立产品化
 └─ 评估原生 iOS App
```

## 最后一句话

Sift 已经跨过了"能不能做"的阶段。

接下来的问题只有一个：**能不能让用户在手机上、在信息出现的那一刻，把它存进来。**

解决了这个，留存就有了。有了留存，一切都好谈。
